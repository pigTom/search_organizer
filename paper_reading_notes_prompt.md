# 论文阅读笔记生成 Prompt

## 核心策略：分段输出，避免卡顿

### 主 Prompt

```
你是一位经验丰富的学术研究助手，擅长阅读和分析学术论文。我需要你帮我阅读一篇论文并生成详细的笔记。

为了避免输出过长导致卡顿，我们将采用分段输出的方式。请严格按照以下流程进行：

## 第一步：论文概览（立即输出）

首先，请提供论文的基本信息和整体结构规划：

1. **论文基本信息**
   - 标题、作者、发表时间、会议/期刊
   - 研究领域和主题

2. **内容结构规划**
   - 列出你将要分析的所有部分（用编号标记）
   - 预估每个部分的重点内容

3. **阅读优先级建议**
   - 哪些部分最重要
   - 建议的阅读顺序

**输出完第一步后，请停止并等待我的指令。**

---

## 后续步骤：逐部分深入分析

在你完成第一步后，我会告诉你继续输出哪个部分。每次只输出一个部分的详细内容。

### 可选的分析部分：

**Part 2: 研究背景与动机**
- 研究问题是什么？
- 为什么这个问题重要？
- 现有方法的局限性
- 本文的创新点预告

**Part 3: 相关工作梳理**
- 主要相关研究分类
- 每类研究的核心思想
- 与本文的关系和区别
- 研究脉络图（可用文字描述）

**Part 4: 方法论详解**
- 核心方法/模型架构
- 关键技术细节
- 算法流程（分步骤说明）
- 创新点深度分析
- 与baseline的对比

**Part 5: 实验设计与结果**
- 实验设置（数据集、评估指标、对比方法）
- 主要实验结果
- 消融实验分析
- 结果的意义和局限性

**Part 6: 批判性思考**
- 方法的优势
- 潜在的问题和局限
- 实验是否充分
- 可能的改进方向

**Part 7: 个人总结与启发**
- 核心贡献总结
- 对我的研究的启发
- 可以借鉴的点
- 值得进一步探索的问题

---

## 使用说明

1. 首次使用时，将论文内容提供给我，并说"请开始第一步"
2. 我完成第一步后会停止，你可以说：
   - "继续Part 2" 或 "继续第2部分"
   - "跳到Part 5" 或 "直接看实验部分"
   - "继续所有剩余部分"（我会逐个输出，每个部分之间会有明确分隔）

3. 如果某个部分内容仍然很长，我会在该部分内部再次分段，并在适当位置停止等待你的"继续"指令

## 输出格式要求

- 使用清晰的Markdown格式
- 重要概念用**粗体**标注
- 公式用LaTeX格式：$inline$ 或 $$block$$
- 使用代码块展示算法伪代码
- 使用列表和表格增强可读性

## 特殊指令

- 如果你想要某个部分更详细，说"Part X 请更详细"
- 如果你想要简化某个部分，说"Part X 简要概括即可"
- 如果你想要我关注特定角度，说"Part X 重点关注[具体方面]"
```

## 使用示例

### 示例对话流程

**用户：**
```
这是论文《Attention Is All You Need》的PDF内容：
[论文内容...]

请开始第一步
```

**AI：**
```
# 论文概览

## 1. 论文基本信息
- **标题**: Attention Is All You Need
- **作者**: Vaswani et al.
- **发表**: NIPS 2017
- **领域**: 自然语言处理、深度学习

## 2. 内容结构规划
我将按以下7个部分进行分析：
- Part 2: 研究背景与动机（序列建模的挑战）
- Part 3: 相关工作（RNN、CNN在序列任务中的应用）
- Part 4: 方法论（Transformer架构详解）
- Part 5: 实验结果（翻译任务性能）
- Part 6: 批判性思考
- Part 7: 个人总结

## 3. 阅读优先级
- 最重要：Part 4（Transformer架构）
- 其次：Part 5（实验验证）
- 建议顺序：按Part 2→4→5→6→7的顺序

**我已完成第一步，请告诉我继续哪个部分。**
```

**用户：** `继续Part 4`

**AI：** [输出Part 4的详细内容...]

## 高级技巧

### 1. 自适应分段

如果某个部分特别复杂（如方法论），可以进一步细分：

```
Part 4 内容较多，我将分为3个子部分：
- Part 4.1: 整体架构
- Part 4.2: Self-Attention机制
- Part 4.3: Position Encoding和其他组件

现在输出Part 4.1...

[内容...]

**Part 4.1完成，输入"继续"查看Part 4.2，或输入"跳到Part 5"**
```

### 2. 检查点机制

在长内容输出中，每隔一定长度设置检查点：

```
[已输出约1000字内容...]

---
**检查点 1/3** - 已完成：架构概述
继续输出请回复"继续"，跳过剩余内容请回复"下一部分"
---
```

### 3. 进度追踪

在每次输出开始时显示进度：

```
📍 当前进度：Part 4/7 - 方法论详解

[内容...]
```

## Prompt 变体

### 简化版（适合快速阅读）

```
请帮我阅读这篇论文并生成笔记。为避免输出卡顿：

1. 首先输出：论文基本信息 + 内容大纲（3-5个要点）
2. 然后停止，等我说"继续第X点"
3. 每次只展开一个要点的详细分析

开始吧！
```

### 深度版（适合精读）

```
请作为我的论文精读助手，采用分段输出策略：

第一步：输出论文地图
- 核心问题、方法、贡献
- 详细内容分为10个部分的规划

后续步骤：我会指定要深入的部分，你每次输出一个部分的：
- 详细解释（假设我是该领域新手）
- 关键公式推导
- 与其他论文的联系
- 批判性分析

每个部分输出后等待我的下一步指令。
```

## 注意事项

1. **明确的停止信号**：每个部分结束时，明确告知用户"已完成XX，等待下一步指令"

2. **灵活的控制**：用户可以随时：
   - 跳过某些部分
   - 要求更详细或更简略
   - 改变输出顺序

3. **内容完整性**：虽然分段输出，但整体内容不减少，只是分批次呈现

4. **上下文保持**：每次继续时，简要回顾前面的内容，保持连贯性

5. **视觉分隔**：使用明显的分隔符（如`---`、`===`）区分不同部分

## 实战建议

### 对于不同类型的论文

**理论型论文**：
- Part 4（方法论）需要更细的分段
- 增加"数学推导"专门部分

**实验型论文**：
- Part 5（实验）可以按实验类型分段
- 增加"实验复现要点"部分

**综述型论文**：
- Part 3（相关工作）是重点，按子领域分段
- 增加"研究趋势分析"部分

### 配合其他工具

- 可以要求AI在每个部分生成思维导图（文字版）
- 可以要求提取关键引用文献列表
- 可以要求生成Anki卡片格式的知识点

## 总结

这个prompt的核心优势：

✅ **避免卡顿**：分段输出，每段长度可控
✅ **内容完整**：不遗漏任何重要信息
✅ **灵活控制**：用户主导阅读节奏
✅ **结构清晰**：模块化设计，易于导航
✅ **可扩展性**：可根据需求调整部分数量和深度

开始使用这个prompt，让论文阅读更高效！
